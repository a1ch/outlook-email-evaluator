Privacy Policy

LAST UPDATED: MARCH 27, 2026

Overview

Clarivise is an AI-powered email security platform consisting of two products: Clarivise Scan (a Chrome extension) and Clarivise Shield (an automated M365 email pipeline). This policy explains what data is collected, how it flows, and how it is protected.

Data Collected During Email Analysis

When an email is analyzed, the following is extracted and sent to our secure analysis service:

  • Email subject line
  • Sender display name and email address
  • Recipient / To line
  • Email body text (up to 3,000 characters)
  • Hyperlink display text and destination domains
  • Attachment file names (if present)
  • Whether the sender was flagged as external by your mail provider

No email content beyond these fields is collected or transmitted. Full message bodies are never stored in our database.

Data Flow

StepWhat happensWho can see email content?
1. Your deviceEmail data is read locallyYou only
2. Network transitData sent over HTTPS (TLS encrypted)No one (encrypted)
3. Supabase Edge FunctionValidates credentials, builds AI promptIn memory only — not stored
4. Anthropic APIAnalyzes email and returns verdictAnthropic (see below)
5. ResponseResult returned and displayedYou only

Anthropic API

Clarivise uses the Anthropic Claude API for email analysis. Per Anthropic's policy, API inputs are not used for model training by default, and data may be retained for up to 30 days for trust and safety purposes.

Scan Logging

Each analysis logs the following for usage reporting: a hashed token identifier, verdict, phishing and spam scores, response time, timestamp, and truncated subject/sender/recipient. Message body text is never written to the scan log.

Data Sharing

Clarivise does not sell, share, or disclose any user data to any third party. Email data is transmitted only to Supabase (infrastructure, data processor) transiently during analysis, and to Anthropic (AI provider) for email analysis.
